# 📝 Contributing Guide – Space Recommendation Project

## 📌 Introduction

Thank you for your interest in contributing to the **Space Recommendation** project!

This project is designed for **open source contribution practice**, so contributions are highly encouraged.
Even small improvements (e.g., fixing typos or improving readability) are valuable.

---

## 📌 Goal of Contribution

The goal of contributing to this project is to:

* Practice open source collaboration
* Improve code quality and readability
* Add new features or enhancements
* Learn GitHub workflow (Issue → Branch → Pull Request)

---

## 📌 Who Can Contribute?

Anyone can contribute, including:

* Beginners in Python
* Students learning open source
* Developers who want to practice collaboration

No advanced experience is required.

---

## 📌 How to Contribute

### 1️⃣ Fork the Repository

Click the **Fork** button on the top right of the repository page.

---

### 2️⃣ Clone Your Fork

```bash
git clone https://github.com/your-username/space-recommendation.git
cd space-recommendation
```

---

### 3️⃣ Create a New Branch

```bash
git checkout -b feature/your-feature-name
```

Use meaningful branch names:

* `feature/add-validation`
* `fix/input-error`
* `docs/update-readme`

---

### 4️⃣ Make Changes

You can contribute by:

#### Code Improvements

* Refactor code for better readability
* Optimize logic

#### Feature Additions

* Add new analysis features
* Improve input validation
* Enhance reporting

#### Documentation

* Improve README
* Add more examples
* Fix typos

---

### 5️⃣ Commit Your Changes

```bash
git add .
git commit -m "Add: short description of your change"
```

Commit message tips:

* Use clear and simple language
* Start with a verb (Add, Fix, Update, Remove)

---

### 6️⃣ Push to GitHub

```bash
git push origin feature/your-feature-name
```

---

### 7️⃣ Create a Pull Request

* Go to your repository on GitHub
* Click **Compare & Pull Request**
* Add a clear description of your changes

---

## 📌 Contribution Guidelines

### Code Style

* Use clear variable names
* Follow consistent formatting
* Add comments where necessary

---

### Input Handling

* Always validate user input
* Avoid crashes due to invalid input

---

### File Structure

* Do not change core structure unnecessarily
* Keep files organized

---

### Documentation

* Update documentation if your change affects usage
* Keep explanations beginner-friendly

---

## 📌 Reporting Issues

If you find a bug or want to suggest a feature:

1. Go to the **Issues** tab
2. Click **New Issue**
3. Provide:

   * Clear title
   * Description
   * Steps to reproduce (if bug)

---

## 📌 What NOT to Do

* Do not submit unrelated changes
* Do not break existing functionality
* Do not use unclear commit messages
* Do not add unnecessary complexity

---

## 📌 Tips for Beginners

* Start with small contributions
* Read the code before modifying
* Ask questions through Issues
* Focus on learning, not perfection

---

## 📌 Contribution Workflow Summary

```text
Fork → Clone → Branch → Edit → Commit → Push → Pull Request
```

---

## 🙆‍♀️ Final Note

Every contribution matters.
This project is a learning space, so feel free to experiment and improve!

Thank you for contributing.
